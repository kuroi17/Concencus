# Profile Picture Upload Debug Plan

## Problem Analysis
The user reports that after selecting an image file for profile picture and clicking "OK", the image is not displaying. The modal closes (indicating save succeeded), but the avatar doesn't show up.

## Potential Issues Identified

### 1. **Upload Success but No Display**
- Image upload to Supabase storage succeeds
- Database update succeeds (modal closes)
- But avatar URL not updating in UI

### 2. **State Synchronization Issues**
- ProfilePage has local `profile` state and `contextProfile` from UserContext
- After upload, both local and global states need to be updated
- Realtime subscription should propagate changes

### 3. **Cache Busting Issues**
- Avatar URLs include timestamp for cache busting
- If cache busting fails, browser may show old cached image

## Debug Steps

### Step 1: Add Console Logging
Add detailed logging to track the upload process:
- File selection in ImageDropzone
- Upload progress in uploadPublicImage
- Database update success/failure
- State updates in ProfilePage

### Step 2: Check Realtime Subscription
Verify the realtime subscription in UserContext is working:
- Check if subscription is active
- Verify payload structure matches expected format
- Ensure bustAvatarCache function is called correctly

### Step 3: Verify Database Updates
Check if the avatar_url is actually being saved to the database:
- Query user_profiles table after upload
- Verify the URL format and accessibility

### Step 4: Test Image Accessibility
Ensure uploaded images are publicly accessible:
- Check Supabase storage bucket permissions
- Verify public URL generation works
- Test image loading in browser

### Step 5: State Management Audit
Review state update logic:
- ProfilePage local state vs UserContext global state
- Timing of updates and potential race conditions
- Ensure all components using avatar display are updated

## Implementation Plan

1. **Add Debug Logging** - Insert console.log statements throughout the upload flow
2. **Fix State Updates** - Ensure proper synchronization between local and global profile states
3. **Improve Error Handling** - Better error messages and user feedback
4. **Test Realtime Updates** - Verify subscription triggers properly
5. **Validate Image URLs** - Ensure generated URLs are correct and accessible

## Expected Outcome
After implementing fixes, profile picture uploads should:
- Successfully upload to Supabase storage
- Update database with correct avatar_url
- Trigger realtime updates across all components
- Display new avatar immediately after upload
- Cache bust properly to show fresh images
